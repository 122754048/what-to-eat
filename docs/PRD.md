# PRD - 产品需求文档「吃什么」(What to Eat)

> 版本: v1.0.0 | 状态: 开发中 | 日期: 2026-03-24

---

## 1. 产品概述

**产品名称**: 「吃什么」(What to Eat)
**产品类型**: iOS 原生应用
**一句话描述**: 用 AI 帮你快速决定今天吃什么，告别选择困难症。

**核心价值**:
- 5 秒内出结果，不给纠结机会
- AI 了解你的心情和场景，生成个性化推荐
- 覆盖全球 8 大菜系 + 健康食谱
- 轻量化，无需注册，打开即用

---

## 2. 用户故事与场景

### 场景 1: 随便决定器（最高频）
> 用户: "中午和同事纠结吃什么，纠结了 20 分钟"

**触发**: 打开 App → 首页
**操作**: 选择心情（5选1）→ 点击"决定了"或甩手机
**结果**: AI 根据心情推荐一道菜 + 大图 + 推荐理由
**行动**: 直接去吃 / 查看食谱 / 换一道 / 收藏

### 场景 2: 特定菜系探索
> 用户: "今晚想吃日料，有什么好推荐？"

**触发**: Tab 2 探索
**操作**: 滑动选择"日餐" → 点击"随机一道"
**结果**: 日料随机菜单 + 详细信息
**行动**: 查看食谱 / 收藏 / 再来一道

### 场景 3: 健康饮食
> 用户: "正在减脂，想吃低卡路里的"

**触发**: Tab 3 健康
**操作**: 选择"减脂"目标 → AI 生成推荐
**结果**: 低卡食谱 + 卡路里 + 营养标签
**行动**: 查看完整食谱 / 跳转电商搜索食材

## 3. 功能规格

### 3.1 Tab 1 - 随便决定器

**心情选择器**:
- 5 个心情选项（横向滚动）:
  - 😋 开心庆祝
  - 😫 疲惫需要安慰
  - 🥂 社交聚会
  - 💪 健身前后
  - 😴 慵懒随便
- 每个心情对应不同的 AI 推荐逻辑

**决策按钮**:
- 位置: 屏幕中央
- 样式: 圆形 120pt，暖橙色 #F4A261
- 文案: "🎲 决定了"
- 动效: 点击时 scale(0.96)，回弹
- 支持: 点击或甩手机两种触发方式

**AI 推荐输出**:
```
{
  "dish_name": "名称",
  "cuisine": "菜系",
  "calories": 450,
  "reason": "推荐理由（15字内）",
  "image_url": "图片URL",
  "recipe": {
    "steps": ["步骤1", "步骤2", "步骤3"],
    "time": "15分钟"
  }
}
```

**结果展示页**:
- 全屏食物大图（占 55% 高度）
- 菜名 + 推荐理由
- 卡路里标签（绿色胶囊）
- 操作按钮: [❤️ 收藏] [🔄 换一道] [📖 食谱]
- 底部 sheet: 简易食谱（3-5步）

### 3.2 Tab 2 - 菜系探索

**8 大菜系**:
| 菜系 | 图标 | 英文 |
|---|---|---|
| 中餐 | 🍜 | Chinese |
| 日餐 | 🍣 | Japanese |
| 韩餐 | 🥘 | Korean |
| 泰餐 | 🍛 | Thai |
| 意餐 | 🍝 | Italian |
| 法餐 | 🥐 | French |
| 美式 | 🍔 | American |
| 印度菜 | 🍛 | Indian |

**交互流程**:
1. 横向滑动选择菜系（选中态: 主色圆形背景）
2. 点击"🎰 随机一道"按钮
3. AI 返回该菜系下的随机菜单

**结果卡片**:
- 左侧大图（60%）
- 右侧: 菜名 + 卡路里 + 简短标签
- 点击展开完整详情

### 3.3 Tab 3 - 健康模式

**4 个健康目标**:
| 目标 | 标签 | AI 逻辑 |
|---|---|---|
| 减脂 | 🍃 低卡路里 | < 400 kcal |
| 增肌 | 💪 高蛋白 | 蛋白质 > 30g |
| 清淡 | 🌿 清爽易消化 | 少油少辣 |
| 低糖 | 🥦 低糖低GI | 碳水 < 30g |

**健康结果卡片**:
- 食物大图
- 菜名
- 营养标签（多个胶囊）: 低卡 / 高蛋白 / 无麸质 等
- 卡路里（大字显示）
- "查看完整食谱"链接

### 3.4 收藏功能
- 点击 ❤️ 保存到本地（UserDefaults）
- 收藏页入口: 每个 Tab 右上角心形图标
- 收藏列表: 本地存储，最多 50 条

### 3.5 食谱详情
- 主要食材列表
- 步骤（3-8步）
- 烹饪时间
- 难度标签（简单/中等）
- 营养信息（热量/蛋白质/碳水/脂肪）

## 4. 技术方案

### 前端
- **框架**: SwiftUI（iOS 17+）
- **图片加载**: AsyncImage + SDWebImageSwiftUI（缓存）
- **动画**: SwiftUI 原生 Animation
- **本地存储**: UserDefaults（收藏 + 设置）

### AI 服务（Cloudflare Workers AI）
- **模型**: `@cf/meta/llama-3.1-8b-instruct`
- **免费额度**: 10,000 请求/天
- **成本**: ~$0.0001/请求

### AI Prompt 设计

**随便决定器 Prompt**:
```
你是一个美食推荐助手。用户心情是[心情]，请推荐一道适合的菜品。
要求：
- 输出JSON格式
- 推荐理由15字内
- 随机选择不同菜系
- 卡路里估算
```

**菜系随机 Prompt**:
```
用户选择了[菜系]，请推荐一道该菜系的经典菜品。
要求：
- 输出JSON格式
- 包含简易做法3步
```

**健康模式 Prompt**:
```
用户目标是[健康目标]，请推荐一道符合的菜品。
要求：
- 输出JSON格式
- 营养信息准确
- 标注适合原因
```
